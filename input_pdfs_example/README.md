# 入力PDFフォルダの例

このフォルダは、実際に使用する際のフォルダ構造の参考例です。

## 実際の使い方

1. プロジェクトルートに `input_pdfs/` フォルダを作成してください
2. 処理したい見開きPDFファイルをそのフォルダに配置
3. `python pipeline.py` を実行
4. `outputs/` フォルダに結果が保存されます

## フォルダ構造の例

```
input_pdfs/          # ← 実際にこのフォルダを作成して使用
├── 2024.pdf
├── 2025.pdf
└── 2026.pdf
```

## 推奨事項

**ファイル名には年度情報を含めてください**

- 推奨例: `2024.pdf`, `2025.pdf`
- ファイル名（拡張子を除く）が自動的にJSONの`year`フィールドに設定されます
- シンプルな年度名にすることで、複数年度のデータを後で比較・分析しやすくなります

## 注意事項

- `input_pdfs/` フォルダは `.gitignore` で除外されているため、PDFファイルはGitリポジトリに含まれません
- PDFファイル名は日本語でもOK（例: `2024年調査.pdf`）
- 複数のPDFを一度に処理できます（すべて自動処理されます）
- 単一ファイルだけ処理したい場合は `python pipeline.py --pdf ".\input_pdfs\ファイル名.pdf"` を使用
